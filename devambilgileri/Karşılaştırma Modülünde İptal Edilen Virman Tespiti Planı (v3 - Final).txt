Tarih Sırası: Orijinal (Borç) işlemin tarihi, iptal (Alacak) işlemin tarihinden KESİNLİKLE KÜÇÜK olmalıdır (aday.IslemTarihi < iptalKaydi.IslemTarihi). Aynı saniyede olan işlemler eşleşmeyecektir.

Zaman Olarak En Yakın: Birden fazla aynı tutarlı aday varsa, iptal işlemine tarih/saat olarak en yakın ve öncesindeki (<) orijinal işlem seçilecek.
Tek Seferlik: IsCancelled == true olan bir kayıt tekrar eşleşemez.
Kelimeler: Yalnızca "iptal", "geri alma", "geri alındı", "düzeltme" kelimeleri taranacak. ("iade" kelimesi çıkarılmıştır).

var iptalEdilenVirmanlar = iptalKayitlari.Where(r => r.Tur == "Virman").ToList(); // Parametreden gelecek veya ComparisonReport üzerinden alınacak
var gecerliVirmanlar = virmanTutarlar; // Parametreden gelen iptal edilmemiş geçerli virman tutarları listesi
decimal toplamGecerliVirman = gecerliVirmanlar.Sum();
if (iptalEdilenVirmanlar.Any() && toplamGecerliVirman >= hedefStopajTutari)
{
    status = StopajStatus.OkWithNote;
    message = $"ℹ️ {iptalEdilenVirmanlar.Sum(x => x.Tutar):N2} ₺ virman iptal edildi, yerine doğru tutarla yeniden yapıldı.";
}
else if (iptalEdilenVirmanlar.Any() && toplamGecerliVirman < hedefStopajTutari)
{
    status = StopajStatus.WarningPending;
    message = $"⚠️ {iptalEdilenVirmanlar.Sum(x => x.Tutar):N2} ₺ virman iptal edildi, ancak yerine yeni virman yapılmamış. Stopaj askıda.";
}
else if (!iptalEdilenVirmanlar.Any() && toplamGecerliVirman < hedefStopajTutari)
{
    status = StopajStatus.Error;
    message = $"❌ Stopaj Hesabına {hedefStopajTutari:N2}₺ Virman bekleniyor, en yakın virman: {toplamGecerliVirman:N2}₺.";
}
5. Test Senaryoları (Edge Cases)
Senaryo A: Aynı tutarda iki ayrı virman + bir iptal → Sadece zaman olarak iptale en yakın öncesindeki (<) orijinal işlem iptal işaretlenecek.
Senaryo B: İptal edilmiş ama yerine yenisi yapılmamış → StopajStatus.WarningPending ile sarı uyarı kutusu görünecek.
Senaryo C: Parçalı virman (10.000 + 6.419,44) → Hepsini toplayacak, StopajStatus.Ok veya duruma göre OkWithNote dönecek.
Senaryo D: İptal + yeni virman + yeni virman da iptal → Tüm karşılık bulan işlemler birbirini sıfırlayacak, WarningPending durumuna düşecek.
