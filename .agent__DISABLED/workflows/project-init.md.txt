WORKFLOW: /project-init

Objective: Build a fast mental model of the repo before coding.

Phase 1 — Repo Map (read-only)
- Identify solution entry points (sln, csproj, package.json, docker).
- List main layers/folders (Web/UI, Application, Domain, Infrastructure, Services, Data).
- Identify configuration files (appsettings*, env, launchSettings, secrets patterns).
- Identify build/run commands from README or scripts.

Phase 2 — Architecture Snapshot
- Summarize the pipeline(s) (data flow, request flow).
- Identify Single Source of Truth locations (core services, engines, DB schema).
- Identify “hot zones” (large controllers/services, critical modules).
- Identify testing approach (unit/integration/e2e) if present.

Phase 3 — Risk Controls
- List locked modules / do-not-touch areas if any.
- Identify migration/DB risks.
- Identify destructive commands to avoid for this repo.

Output format:
1) Repo Map (bullets)
2) Architecture Snapshot (bullets)
3) Risk Controls (bullets)
4) Recommended next step: ask for the minimum task input (use /task-template)
