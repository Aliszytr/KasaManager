# KasaManagerProject vNext — FULL CONTEXT & STATUS REPORT

## 1. PROJENİN ANA AMACI

Bu projenin amacı:

* Kasa hesaplama sistemini **deterministik**, **şeffaf** ve **tek kaynaklı (SSOT)** hale getirmek
* Legacy (eski) karmaşık ve hataya açık hesaplama zincirlerini kaldırmak
* Snapshot bağımlılığını ortadan kaldırarak sistemi:

  * Debug edilebilir
  * Test edilebilir
  * Stabil
  * Production-ready hale getirmek

---

## 2. BAŞLANGIÇTAKİ PROBLEM

Sistemde şu problemler vardı:

### ❌ Snapshot bağımlılığı

* Hesaplama sonucu snapshot’a yazılıyor
* Sonraki hesaplamalar snapshot’tan okunuyordu
* Bu durum:

  * Gizli state
  * Tutarsız sonuçlar
  * Debug zorluğu yaratıyordu

### ❌ Legacy EksikFazla Chain

* Recursive yapı
* Çok katmanlı fallback
* Karmaşık ve izlenemez
* Performans problemi

### ❌ Parity problemi

* Yeni sistem ile eski sistem farklı sonuçlar üretebiliyordu

---

## 3. HEDEF MİMARİ

Hedef:

```
Input (Excel / User Data)
   ↓
Unified Pool (SSOT)
   ↓
Projection Engine (TEK hesap motoru)
   ↓
Result (UI)
```

Kurallar:

* Tek hesap motoru → Projection Engine
* Tek veri kaynağı → Unified Pool
* Snapshot → aktif sistem DIŞI
* Deterministik → aynı input = aynı output
